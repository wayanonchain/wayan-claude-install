2. Что это за система

Wayan Agents — это конструктор личных AI-агентов. Вы арендуете небольшой сервер, запускаете на нём установщик из открытого репозитория — и получаете двух агентов в Telegram. Каждая часть системы делает одну понятную вещь:

ЧастьРоль простыми словами
JupiterГлавный агент: исследования, анализ, контент. Ему вы пишете каждый день.
UranЗапасной «техник»: диагностика сервера, логи, перезапуск Jupiter, если тот упал. Отдельный бот — независимый канал связи.
TelegramИнтерфейс. Никаких сайтов и консолей в быту — вы просто пишете боту (текст, голос, файлы, видео).
VPSАрендованный сервер, где агенты живут 24/7 — даже когда ваш компьютер выключен.
Claude Code«Мозг»: модель Claude с инструментами — думает, выполняет команды, читает и пишет файлы.
GitHubИсточник истины: код, шаблоны, правила и история всех изменений. Откуда ставим и куда коммитим.
OpenVikingДолговременная семантическая память: агент находит нужные знания по смыслу, а не по точному слову.
MarkdownФормат всех знаний: обычные текстовые файлы, которые читают и люди, и агенты.
ObsidianУдобное окно в эти Markdown-знания для человека: читать, править, наводить порядок.
GroqРаспознавание речи (Whisper): голосовые и аудиодорожки видео превращаются в текст.
OpenAIПровайдер эмбеддингов и VLM для OpenViking — то, что делает память «смысловой».
ffmpegИзвлекает кадры из видео, чтобы Claude мог «посмотреть» ролик глазами.
WAYAN_PACKПресет профессии при установке: onchain-аналитик, креатор, devops, студент, фаундер.
Почему два агента, а не один? Один агент — единая точка отказа: если он завис, никто его удалённо не поднимет. Uran — независимый второй канал со своим ботом и своим сервисом, с ограниченным sudo ровно на «посмотреть логи и перезапустить Jupiter». Jupiter упал → пишете Uran → Uran чинит.

3. Архитектура: три схемы

3.1 Общий поток: от сообщения до знаний

Пользователь ↓ Telegram (текст · голос · файлы · видео) ↓ Jupiter / Uran (gateway + очередь задач) ↓ Claude Code (мозг: думает и действует) ↓ Skills · Rules · Memory · OpenViking ↓ Markdown → Git → Obsidian

Каждое сообщение проходит этот путь: Telegram принимает, gateway ставит задачу в очередь, Claude Code решает её, опираясь на навыки (skills), правила (rules) и память, а результат оседает в Markdown-файлах, которые версионируются в Git и читаются в Obsidian.

3.2 Поток видео: визуальный анализ

Telegram-видео ↓ скачивание во временную папку ↓ Groq: аудио → текст + ffmpeg: ключевые кадры └──────────────┬───────────────┘ ↓ Claude: смотрит кадры + читает текст ↓ единый разбор: ## Аудио / ## Видео / ## Вместе ↓ Markdown-транскрипт + ответ в Telegram

Видео анализируется по двум каналам сразу: Groq расшифровывает звук, ffmpeg вытаскивает несколько равномерных кадров, Claude смотрит каждый кадр и сводит всё в один ответ. Кадры и исходное видео — временные: после ответа остаётся только Markdown-транскрипт.

3.3 Поток обновлений: repo-first деплой

GitHub-репозиторий (источник истины) ↓ scripts/update.sh (git pull + тесты + проверка дрейфа) ↓ тесты прошли ↓ scripts/deploy-gateway.sh (бэкап → синхронизация) ↓ /opt/wayan-jupiter + /opt/wayan-uran ↓ systemd restart (перезапуск сервисов с проверкой)

Код никогда не правится прямо на сервере. Изменение → коммит в GitHub → скрипт обновления с тестами → деплой с автоматическим бэкапом. Подробно — в разделе 16.

4. Где что живёт на сервере

После установки на VPS появляется несколько ключевых путей. Понимать их важно: половина проблем новичков — «правил не тот файл не в том месте».

ПутьЧто это
/root/wayan-claude-installКлон GitHub-репозитория на сервере. Рабочая копия источника истины: отсюда обновляемся и деплоим.
/opt/wayan-jupiter/gatewayПродакшн-код Jupiter: Telegram-gateway + Python-окружение. Сюда код попадает только через деплой-скрипт.
/opt/wayan-uran/gatewayТо же самое для Uran — отдельная копия, чтобы агенты были независимы.
/home/wayan/.claude-lab/jupiterРабочее пространство Jupiter: CLAUDE.md (роль), skills/, orchestration/, transcripts/, uploads/, PACK.md (если выбран пакет).
/home/wayan/.claude-lab/uranРабочее пространство Uran — та же структура.
/home/wayan/.openvikingПамять OpenViking: конфиг ov.conf (с ключами, права 600) и данные индекса. В Git не попадает никогда.
/etc/wayan-jupiter.envСекреты и настройки Jupiter: токен бота, ключ Groq, флаги видео/голоса. Права 600/640.
/etc/wayan-uran.envТо же для Uran — отдельный токен, отдельный файл.
/opt/wayan_pirat_botЛегаси-бот, не часть этого репозитория. Не трогать: у него своя жизнь и свои таймеры.

Репозиторий vs продакшн: почему это разные места

Репозиторий (/root/wayan-claude-install) — это «чертежи»: код с историей, тестами и возможностью отката. Продакшн (/opt/wayan-*) — это «работающий станок»: то, что реально запущено сервисами systemd.

Почему Git — источник истины. Если истина в Git, любую машину можно восстановить с нуля, каждое изменение видно в истории, и тесты прогоняются до того, как код попадёт в продакшн.

Почему править /opt напрямую опасно. Такая правка не видна в Git («дрейф»): следующий деплой её молча перезапишет, откатиться некуда, а почему «вчера работало» — уже не выяснить. Был реальный случай дрейфа в этом проекте — теперь его ловит проверка deploy-gateway.sh --check.

6. Словарь новичка

33 термина, которые встретятся в гиде. Каждый блок раскрывается: что это, зачем, аналогия, типичная ошибка и пример из Wayan Agents.

SSH

Что это: протокол защищённого подключения к удалённому компьютеру через терминал.

Зачем: весь сервер управляется через SSH — установка, настройка, логи.

Аналогия: телепорт в комнату с сервером: вы «сидите за его клавиатурой», не вставая со стула.

Типичная ошибка: путать, где вы сейчас — в локальном терминале или уже на сервере. Смотрите на приглашение строки: root@server = вы на сервере.

В Wayan: ssh root@SERVER_IP — первый шаг установки.

VPS

Что это: Virtual Private Server — арендованный виртуальный сервер в дата-центре.

Зачем: агенты должны работать 24/7, а ваш ноутбук — нет.

Аналогия: съёмная квартира для ваших программ: маленькая, зато всегда «дома».

Типичная ошибка: взять 512 МБ RAM «подешевле» — память и агенты начнут задыхаться. Берите 2 ГБ.

В Wayan: вся система живёт на одном VPS с Ubuntu 22.04/24.04.

Linux-пользователь

Что это: учётная запись внутри сервера со своими файлами и правами.

Зачем: разграничение: что может делать программа, запущенная от этого имени.

Аналогия: бейджи в офисе — у уборщика и директора разные двери открываются.

Типичная ошибка: выполнять всё под root, потому что «так проще». Так одна ошибка сносит весь сервер.

В Wayan: установщик создаёт пользователя wayan — агенты работают от него, не от root.

root

Что это: суперпользователь Linux — может всё, без вопросов.

Зачем: установка пакетов, создание сервисов — это нужно делать от root (через sudo).

Аналогия: мастер-ключ от всего здания.

Типичная ошибка: запускать от root то, что должно работать от обычного пользователя (например, логин Claude).

В Wayan: install.sh требует root; но в Claude логинимся как wayan: sudo -u wayan -i.

Git

Что это: система контроля версий — запоминает каждое состояние файлов.

Зачем: история изменений, откат, совместная работа, «источник истины».

Аналогия: сохранения в игре: в любой момент можно вернуться к прошлому «сейву».

Типичная ошибка: править файлы на сервере мимо Git — изменения потеряются при следующем деплое.

В Wayan: весь код и шаблоны живут в Git; обновление = git pull + тесты + деплой.

GitHub

Что это: сайт-хранилище Git-репозиториев.

Зачем: репозиторий доступен с любой машины; можно форкать чужие проекты.

Аналогия: облачный диск, но для кода и с полной историей.

Типичная ошибка: закоммитить секрет в публичный репозиторий. Ключ мгновенно скомпрометирован — отзывайте.

В Wayan: установка идёт прямо из github.com/wayanonchain/wayan-claude-install.

Репозиторий

Что это: папка проекта под управлением Git: файлы + вся их история.

Зачем: единица работы: клонируете репозиторий — получаете весь проект.

Аналогия: папка-скоросшиватель со всеми черновиками всех версий документа.

Типичная ошибка: запускать команды установки не из папки репозитория.

В Wayan: на сервере репозиторий лежит в /root/wayan-claude-install.

Коммит (commit)

Что это: зафиксированный «снимок» изменений с подписью, что и зачем поменяли.

Зачем: каждый шаг истории имеет имя и может быть откатан.

Аналогия: запись в бортовом журнале: дата, что сделано, кем.

Типичная ошибка: один гигантский коммит «поправил всё». Мелкие осмысленные коммиты легче откатывать.

В Wayan: «обучение» агента — это тоже коммиты: новые правила и знания попадают в историю.

Push / Pull

Что это: push — отправить свои коммиты на GitHub; pull — забрать чужие/новые себе.

Зачем: так локальная копия и GitHub остаются синхронными.

Аналогия: отправка и получение посылок между вашим столом и складом.

Типичная ошибка: забыть pull на сервере и удивляться, что обновлений «нет».

В Wayan: scripts/update.sh делает pull + тесты за вас.

Ветка (branch)

Что это: параллельная линия истории в репозитории.

Зачем: экспериментировать, не ломая основную версию (main).

Аналогия: черновик отдельно от чистовика: дописали, проверили — перенесли.

Типичная ошибка: месяцами жить в своей ветке и копить конфликты с main.

В Wayan: установка и деплой идут из ветки main.

Docker

Что это: запуск программ в изолированных «контейнерах» со всем их окружением.

Зачем: программа везёт всё своё с собой — не конфликтует с системой.

Аналогия: еда в контейнере: разогрел где угодно, посуда не нужна.

Типичная ошибка: открыть порт контейнера на весь интернет (0.0.0.0) вместо localhost.

В Wayan: OpenViking работает в Docker и слушает только 127.0.0.1:1933.

systemd

Что это: менеджер служб Linux: запускает программы при старте и перезапускает упавшие.

Зачем: агенты должны переживать перезагрузку сервера без вашего участия.

Аналогия: дежурный по зданию: свет включает, упавшие автоматы поднимает.

Типичная ошибка: запустить программу руками в терминале и закрыть его — программа умрёт вместе с сессией.

В Wayan: wayan-jupiter.service и wayan-uran.service управляются systemd.

Сервис (service)

Что это: программа под управлением systemd: фоновая, автозапускаемая, наблюдаемая.

Зачем: единые команды: start / stop / restart / status + логи в journald.

Аналогия: бытовой прибор на розетке с автоматом: выбило — автомат вернул.

Типичная ошибка: после правки env-файла забыть systemctl restart — настройки читаются при старте.

В Wayan: systemctl status wayan-jupiter.service — первое, что проверяется при проблемах.

Переменная окружения

Что это: именованное значение, которое программа читает при запуске (KEY=value).

Зачем: настройки и секреты живут вне кода — код публичен, значения приватны.

Аналогия: стикеры на холодильнике для домработницы: инструкции отдельно от самой работы.

Типичная ошибка: кавычки/пробелы вокруг значения (TOKEN = "abc") — пишите TOKEN=abc.

В Wayan: все настройки агентов — в /etc/wayan-jupiter.env и /etc/wayan-uran.env.

API-ключ

Что это: секретная строка, по которой сервис узнаёт вас и списывает деньги.

Зачем: Groq и OpenAI работают только по ключу.

Аналогия: банковская карта: кто держит в руках — тот и платит. Не «пароль», а именно карта.

Типичная ошибка: вставить ключ в чат/скриншот/GitHub. Утёк — отзывайте немедленно.

В Wayan: ключи лежат только в env-файлах (права 600) и ov.conf — никогда в Git.

Telegram-бот

Что это: аккаунт-программа в Telegram, которой управляет ваш сервер.

Зачем: готовый интерфейс: чат, голосовые, файлы — без разработки приложения.

Аналогия: ресепшн вашей системы: всё общение через него.

Типичная ошибка: использовать один токен для двух агентов — Uran перестаёт быть независимым каналом.

В Wayan: два бота: один для Jupiter, один для Uran, токены в разных env-файлах.

BotFather

Что это: официальный бот Telegram, который создаёт других ботов.

Зачем: единственный легальный способ получить токен бота.

Аналогия: паспортный стол для ботов.

Типичная ошибка: нарваться на фейкового «BotFather». Настоящий — @BotFather с синей галочкой.

В Wayan: /newbot дважды — для Jupiter и для Uran.

Claude Code

Что это: агентная среда Anthropic: модель Claude + инструменты (терминал, файлы) в CLI.

Зачем: это и есть «мозг» агентов — он не только отвечает, но и делает.

Аналогия: не «справочник», а стажёр с допуском к компьютеру: читает, пишет, запускает.

Типичная ошибка: логиниться под root. Логин нужен под пользователем wayan — от него работают сервисы.

В Wayan: gateway вызывает claude -p на каждую задачу из Telegram.

Groq

Что это: сервис сверхбыстрого инференса; нам нужен его Whisper — распознавание речи.

Зачем: голосовые и звук видео превращаются в текст за секунды.

Аналогия: стенографист, который успевает за час аудио — за полминуты.

Типичная ошибка: положить ключ Groq не в тот env (нужен в оба, если голос нужен обоим агентам).

В Wayan: GROQ_API_KEY в env; модель — whisper-large-v3-turbo.

Эмбеддинги OpenAI

Что это: превращение текста в вектор чисел, где близкий смысл = близкие векторы.

Зачем: так память ищет «похожее по смыслу», а не по совпадению слов.

Аналогия: координаты на «карте смыслов»: соседние точки — родственные мысли.

Типичная ошибка: не пополнить баланс OpenAI — память молча не сможет сохранять.

В Wayan: OpenViking использует text-embedding-3-small + gpt-4o-mini.

OpenViking

Что это: сервер семантической памяти, работает в Docker на вашем VPS.

Зачем: долговременная память: «найди в памяти похожие кейсы» работает через него.

Аналогия: библиотекарь, помнящий каждую книгу по содержанию, а не по корешку.

Типичная ошибка: открыть его порт наружу. Он должен слушать только localhost.

В Wayan: 127.0.0.1:1933, общая память Jupiter и Uran, данные в ~/.openviking.

Семантическая память

Что это: память, отвечающая на «о чём это было?», а не «где точная фраза?».

Зачем: агент вспоминает релевантное, не перечитывая все файлы каждый раз.

Аналогия: человеческая память: вы помните суть разговора, а не стенограмму.

Типичная ошибка: сваливать в память всё подряд — мусор на входе, мусор на выходе.

В Wayan: хранится короткое и важное; источник истины остаётся в Markdown+Git.

Markdown

Что это: текстовый формат с простой разметкой: # заголовок, - список, **жирный**.

Зачем: читается человеком, агентом и Git одинаково хорошо; не зависит ни от каких программ.

Аналогия: бумага и ручка цифрового мира: переживёт любые модные приложения.

Типичная ошибка: хранить знания в проприетарных форматах — агент их не прочитает.

В Wayan: правила, навыки, память, транскрипты — всё это Markdown-файлы.

Obsidian

Что это: бесплатное приложение для работы с папкой Markdown-заметок (vault).

Зачем: человеку удобно читать, править и связывать знания агентов.

Аналогия: читальный зал над архивом: документы те же, работать приятнее.

Типичная ошибка: думать, что Obsidian «хранит» данные. Он только показывает обычные файлы.

В Wayan: открываете папку знаний как vault — и видите всё, что накопили агенты.

ffmpeg

Что это: универсальный конвертер аудио/видео для командной строки.

Зачем: вытаскивает из видео ключевые кадры, чтобы Claude мог их «увидеть».

Аналогия: швейцарский нож для медиа: порезать, сжать, переконвертировать.

Типичная ошибка: включить визуальный анализ видео, не установив ffmpeg — агент честно скажет об этом.

В Wayan: VIDEO_VISUAL_ANALYSIS=true + ffmpeg = видео анализируется и по картинке.

Webhook

Что это: способ доставки событий: сервис сам стучится HTTP-запросом на ваш адрес.

Зачем: мгновенная доставка, но нужен открытый порт — а это поверхность атаки.

Аналогия: курьер звонит в вашу дверь — удобно, но дверь должна быть наружу.

Типичная ошибка: оставить порт вебхука открытым всему интернету.

В Wayan: агенты вебхуки не используют (см. polling). У легаси-бота вебхук-порт 8080 был публичным — закрыт в фазе M6.

Polling

Что это: программа сама периодически спрашивает сервер: «есть новые сообщения?»

Зачем: не нужно открывать никакие порты — исходящие запросы безопаснее входящих.

Аналогия: сами ходите проверять почтовый ящик, вместо того чтобы давать курьеру ключ от двери.

Типичная ошибка: думать, что polling «медленный» — long polling отдаёт сообщения почти мгновенно.

В Wayan: оба агента получают сообщения Telegram через long polling. Открытых портов — ноль.

Firewall (файрвол)

Что это: фильтр сетевых соединений: какие порты открыты, какие закрыты.

Зачем: всё, что не должно быть доступно из интернета, должно быть закрыто.

Аналогия: охрана на входе: пускает только тех, кто в списке.

Типичная ошибка: открыть порт «на минутку для теста» и забыть закрыть.

В Wayan: ufw: открыт по сути только SSH; порт 8080 легаси-бота из правил удалён.

No-clobber

Что это: режим копирования «не перезаписывать существующее» (cp -n).

Зачем: повторная установка не затирает ваши правки шаблонов, правил и памяти.

Аналогия: ремонтник доносит недостающую мебель, но не трогает ту, что вы уже расставили.

Типичная ошибка: ждать, что переустановка «обновит» отредактированный файл — нет, ваши правки в приоритете.

В Wayan: установщик и пакеты WAYAN_PACK копируют всё только no-clobber.

Деплой (deploy)

Что это: доставка новой версии кода в продакшн.

Зачем: чтобы «обновить агентов» было одной проверяемой процедурой, а не ручными правками.

Аналогия: выпуск нового тиража книги: проверили вёрстку — заменили на полках.

Типичная ошибка: «задеплоить» правкой файла прямо на сервере. Это и есть дрейф.

В Wayan: scripts/deploy-gateway.sh: бэкап → синхронизация → проверка → рестарт.

Откат (rollback)

Что это: возврат к предыдущей рабочей версии после неудачного обновления.

Зачем: любое обновление может сломаться — должен быть путь назад за минуты.

Аналогия: «отменить» в редакторе, только для всего сервера.

Типичная ошибка: не иметь бэкапа и чинить сломанное «вперёд», в панике.

В Wayan: каждый деплой оставляет /opt/wayan-*/gateway.bak.ДАТА — откат = вернуть папку и перезапустить.

WAYAN_PACK

Что это: переменная окружения, выбирающая профессиональный пакет при установке.

Зачем: один установщик — разные специализации: onchain, creator, devops, student, founder.

Аналогия: комплектация автомобиля: база одна, опции под ваш сценарий.

Типичная ошибка: забыть -E в sudo -E — sudo «съест» переменную, и пакет не применится.

В Wayan: WAYAN_PACK=creator sudo -E ./install.sh. Без переменной — обычная установка.

Дрейф (drift)

Что это: расхождение между кодом в Git и кодом, реально работающим в продакшне.

Зачем знать: дрейф — тихий враг: «вчера работало», а почему — никто не знает.

Аналогия: чертёж говорит одно, а в станке уже наварено другое.

Типичная ошибка: «быстро поправлю на сервере, потом перенесу в Git» — и не перенести.

В Wayan: bash scripts/deploy-gateway.sh --check — отчёт PASS/FAIL по дрейфу (только имена файлов).

7. Три пути установки

Путь A. С Mac (или любого компьютера) на свой VPS — рекомендуем новичкам

Ваш компьютер — пульт управления; всё ставится на сервер. Сначала проверьте инструменты (обе команды уже встроены в macOS; на Windows подойдёт PowerShell или WSL):

git --version
ssh -V

Подключитесь к серверу (IP и пароль выдаёт VPS-провайдер):

ssh root@SERVER_IP

Чтобы не вводить пароль каждый раз — создайте SSH-ключ. Ключ принадлежит вам (вашему компьютеру-клиенту), а не Wayan и не серверу: приватная половина никогда не покидает ваш ноутбук, на сервер копируется только публичная.

ssh-keygen -t ed25519 -C "my-wayan-key"
ssh-copy-id root@SERVER_IP

И добавьте удобный псевдоним в ~/.ssh/config:

Host wayan-vps HostName SERVER_IP User root IdentityFile ~/.ssh/id_ed25519

Теперь подключение — просто ssh wayan-vps.

Cursor / VS Code Remote-SSH. Если поставите расширение Remote-SSH (или используете Cursor), сможете открывать файлы сервера как локальные — удобно для правки шаблонов и чтения логов. Это опционально: всё в гиде делается и в чистом терминале.

Путь B. Прямо на VPS

Если у провайдера есть веб-консоль (браузерный терминал) — можно вообще без SSH с компьютера: открываете консоль в личном кабинете провайдера и выполняете те же команды установки из раздела 8, начиная с шага 4. Минус: консоль в браузере неудобнее для долгой работы — SSH-путь A стоит освоить позже в любом случае.

Путь C. Форк и кастомизация репозитория — для разработчиков

  1. Сделайте fork репозитория wayanonchain/wayan-claude-install на GitHub.
  2. Перепишите шаблоны под себя: templates/jupiter/CLAUDE.md, templates/uran/CLAUDE.md, USER.md, skills.
  3. Добавьте или измените пакеты в packs/ (см. раздел 9 и docs/PACKS.md).
  4. Ставьте уже из своего форка: в install.sh поменяйте REPO_URL или клонируйте форк и запускайте локально.
  5. Никогда не коммитьте секреты: env-файлы, токены, ov.conf, accounts.md — всё это уже в .gitignore, не обходите его.

8. Установка шаг за шагом

18 шагов от нуля до полностью проверенной системы. Для каждого: команда, смысл, ожидаемый результат и типичная ошибка. Не спешите: каждый шаг проверяется до перехода к следующему.

Шаг 1. Аккаунт GitHub

Что: зарегистрируйтесь на github.com. Зачем: отсюда ставится система; сюда позже поедет ваша база знаний и ваш форк.

Проверка: вы можете открыть github.com/wayanonchain/wayan-claude-install и видите код. Типичная ошибка: регистрироваться «потом» — без GitHub не будет пути C и экспорта знаний.

Шаг 2. Создайте VPS

Что: у любого провайдера создайте сервер: Ubuntu 22.04 или 24.04, минимум 1 ГБ, рекомендуем 2 ГБ RAM. Запишите IP и root-пароль.

Зачем: это дом агентов. Проверка: сервер в статусе «running», IP известен.

Типичная ошибка: выбрать другую ОС (Debian, CentOS) — установщик поддерживает только Ubuntu 22.04/24.04 и честно откажется.

Шаг 3. Подключитесь по SSH

ssh root@SERVER_IP

Что: заходите на сервер. При первом подключении согласитесь с fingerprint (yes), введите пароль.

Ожидаемо: приглашение вида root@hostname:~# — теперь команды выполняются на сервере.

Типичная ошибка: Permission denied — опечатка в пароле или IP; «connection timed out» — сервер ещё создаётся или IP неверный.

Шаг 4. Установите Wayan Agents

Вариант 1 — одной командой (быстрее):

curl -fsSL https://raw.githubusercontent.com/wayanonchain/wayan-claude-install/main/install.sh | sudo bash

Вариант 2 — из клона (видно, что запускаете; нужен для пакетов WAYAN_PACK):

git clone https://github.com/wayanonchain/wayan-claude-install.git cd wayan-claude-install sudo ./install.sh

Что происходит: ставятся пакеты, Node.js, Claude Code; создаётся пользователь wayan; разворачиваются рабочие пространства, gateway, сервисы systemd. Установщик идемпотентен — повторный запуск ничего не ломает и не затирает ваши правки (no-clobber).

Ожидаемо: зелёный финальный баннер «installation complete» с разделом NEXT STEPS.

Типичная ошибка: запуск без sudo → «This installer must run as root». Хотите пакет профессии — см. раздел 9 до этого шага.

Шаг 5. Создайте двух ботов у BotFather

Что: в Telegram откройте @BotFather → /newbot → имя и username для Jupiter; повторите для Uran. Сохраните оба токена (формат 123456:ABC-DEF...).

Зачем два: Uran — независимый канал; если бот Jupiter недоступен, вы всё ещё на связи с сервером.

Типичная ошибка: перепутать токены местами — тогда «Jupiter» будет отвечать в чате Uran и наоборот.

Шаг 6. Пропишите токены в env-файлы

sudoedit /etc/wayan-jupiter.env

Найдите строку и вставьте токен (без кавычек и пробелов):

TELEGRAM_BOT_TOKEN=YOUR_BOT_TOKEN
sudoedit /etc/wayan-uran.env

Зачем: gateway читает токен из env при старте — это единственное место, где он хранится.

Типичная ошибка: отредактировать и забыть перезапустить сервис (шаг 12) — env читается только при старте.

Шаг 7. Войдите в Claude Code от пользователя wayan

sudo -u wayan -i claude

Что: станьте пользователем wayan и запустите claude — пройдите логин по ссылке в браузере (нужна подписка Pro/Max). После входа выйдите из claude и обратно в root: exit.

Зачем: сервисы работают от wayan — логин root им не виден.

Ожидаемо: claude --version работает; повторный запуск claude не просит логин.

Типичная ошибка: залогиниться под root: агенты будут отвечать ошибкой на каждое сообщение.

Шаг 8. Добавьте ключ Groq (голос)

sudoedit /etc/wayan-jupiter.env
GROQ_API_KEY=YOUR_API_KEY

Повторите для /etc/wayan-uran.env, если голос нужен и Uran.

Зачем: без ключа голосовые сообщения вежливо отклоняются, текст работает как обычно.

Типичная ошибка: создать ключ, но не проверить лимиты/баланс в console.groq.com.

Шаг 9. Ключ OpenAI для памяти

Что: создайте ключ на platform.openai.com и пополните баланс минимум на $5 — без денег эмбеддинги не работают и память не сможет сохранять.

Куда: ключ пойдёт в конфиг OpenViking /home/wayan/.openviking/ov.conf (следующий шаг) — не в env-файлы агентов и тем более не в Git.

Типичная ошибка: ключ есть, баланс 0 → memory_store будет падать с невнятной ошибкой квоты.

Шаг 10. Установите и запустите OpenViking

Что: OpenViking разворачивается в Docker по инструкции docs/OPENVIKING_MEMORY.md: конфиг ov.conf (ключ OpenAI + локальный root_api_key, права 600, владелец wayan) и docker compose в /home/wayan/.openviking.

cd /home/wayan/.openviking && docker compose up -d

Критично: порт публикуется только как 127.0.0.1:1933 — никогда 0.0.0.0. Проверка:

curl -s http://127.0.0.1:1933/health

Ожидаемо: ответ со статусом «здоров». Типичная ошибка: нет Docker (docker: command not found) — поставьте по официальной инструкции docs.docker.com; или порт открыт наружу — проверьте docker ps: в PORTS должно быть 127.0.0.1:1933.

Шаг 11. Подключите память к Claude (MCP-инструменты)

Что: по docs/OPENVIKING_MEMORY.md подключите MCP-инструменты памяти к Claude Code пользователя wayan — у агентов появятся memory_store, memory_recall, memory_health.

Зачем: это «мост» между мозгом (Claude) и памятью (OpenViking). Разрешения на эти инструменты уже прописаны в профилях прав агентов.

Типичная ошибка: подключить под root, а не под wayan — агенты инструментов не увидят.

Шаг 12. Запустите и проверьте Jupiter

systemctl start wayan-jupiter.service systemctl status wayan-jupiter.service

Ожидаемо: active (running). Напишите боту Jupiter в Telegram «привет» — придёт мгновенный ак «✅ Task received. Queue position: 1», затем ответ.

Типичная ошибка: молчит → токен не тот или сервис не перезапущен после правки env: systemctl restart wayan-jupiter.service, логи: journalctl -u wayan-jupiter.service -e.

Шаг 13. Запустите и проверьте Uran

systemctl start wayan-uran.service systemctl status wayan-uran.service

Напишите боту Uran: «Проверь состояние сервера» — он должен ответить диагностикой.

Зачем сейчас: Uran — ваша страховка; проверяйте его до того, как он впервые понадобится.

Шаг 14. Проверьте память: store и recall

Напишите Jupiter: «Запомни: мой часовой пояс — Бали (UTC+8)» — агент сохранит через memory_store. Затем: «Что ты помнишь о моём часовом поясе?» — должен вспомнить через memory_recall. Бонус-проверка: спросите то же у Uran — память общая, он тоже должен знать.

Типичная ошибка: ошибка квоты OpenAI (см. шаг 9) или OpenViking не запущен (шаг 10).

Шаг 15. Проверьте голос

Отправьте Jupiter голосовое сообщение. Ожидаемо: придёт расшифровка (📝 …), затем обычный ответ по сути сказанного.

Типичная ошибка: «voice is disabled» → нет GROQ_API_KEY в env или сервис не перезапущен.

Шаг 16. Проверьте PDF

Отправьте PDF-файл с подписью «Сделай summary». Ожидаемо: структурированный разбор, а в transcripts/ рабочего пространства появится Markdown-транскрипт.

Типичная ошибка: файл больше ~20 МБ — лимит Telegram для ботов; агент сам предложит прислать прямую ссылку.

Шаг 17. Проверьте визуальный анализ видео

Включите функцию (по умолчанию выключена) и убедитесь, что есть ffmpeg:

apt-get install -y ffmpeg sudoedit /etc/wayan-jupiter.env # VIDEO_VISUAL_ANALYSIS=true systemctl restart wayan-jupiter.service

Отправьте короткое видео с подписью «Проанализируй: что видно и что сказано». Ожидаемо: «🎥 Video received…», затем единый ответ с блоками «Аудио / Видео / Вместе».

Типичная ошибка: ffmpeg не установлен — агент явно скажет и сделает разбор только по звуку (без «тихих» отказов).

Шаг 18. Проверьте repo-first деплой

cd /root/wayan-claude-install bash scripts/update.sh bash scripts/deploy-gateway.sh --check

Ожидаемо: тесты зелёные, drift-отчёт — PASS (продакшн совпадает с Git).

Зачем: вы только что проверили главный механизм обслуживания системы. Если PASS — установка завершена правильно.

✅ Всё прошло? Поздравляем — система собрана. Дальше: выберите пакет профессии, поймите память и научите агентов работать по-вашему. Быстрая самопроверка всей системы: sudo bash scripts/healthcheck.sh.

9. WAYAN_PACK: пакеты профессий

Пакет — это пресет профессии, выбираемый при установке. Технически — набор Markdown-файлов, которые накладываются поверх обычной установки: акцент роли (PACK.md), заготовка памяти, правила маршрутизации задач и карта сервисов. Пакеты существуют, чтобы агент с первого дня «думал» в вашей профессии, а не настраивался неделями.

Три гарантии пакетов. 1) Это только Markdown — никаких новых сервисов, ключей или прав. 2) No-clobber: пакет никогда не перезаписывает существующие файлы — ваши правки в приоритете. 3) Без WAYAN_PACK установка идентична обычной — пакеты ни на что не влияют, пока вы их не попросите.
sudo ./install.sh
WAYAN_PACK=onchain sudo -E ./install.sh
WAYAN_PACK=creator sudo -E ./install.sh
WAYAN_PACK=devops sudo -E ./install.sh
WAYAN_PACK=student sudo -E ./install.sh
WAYAN_PACK=founder sudo -E ./install.sh

Флаг -E обязателен: он передаёт переменную через sudo. Ошибка в имени пакета безопасна — установщик откажется до любых изменений и покажет список доступных пакетов.

Трейдер, аналитик, владелец альфа-канала. Это «родной» профиль Wayan, оформленный как пакет.

Jupiter: разбор токенов и smart money, вердикты watch/avoid/deep dive, треды из ресёрча.
Uran: здоровье пайплайна данных и сервера.

Примеры: «разбери токен X», «кто из smart money заходил за неделю», «сделай X-thread из этого разбора».

🎬 creator — контент-мейкер

Примеры: голосовая идея → сценарий Reels, «перепакуй это видео в 5 коротких», контент-план недели.

Соло-дев или маленькая команда со своими сервисами.

11. Зачем здесь Obsidian

Простая идея: агенты пишут Markdown — Obsidian его красиво показывает человеку. Никакой магии: Obsidian просто открывает ту же папку файлов, которую пишут агенты и хранит Git.

  • Obsidian — для людей. Агентам он не нужен: они читают файлы напрямую.
  • Git хранит те же файлы с историей; вы можете спокойно править — изменения коммитятся.
  • Вы просматриваете и редактируете заметки, наводите связи, чистите устаревшее.
  • Потом — commit в Git: правки становятся официальной версией знаний.
Голос / видео / PDF ↓ Markdown-транскрипт (агент) ↓ Вы читаете и правите в Obsidian ↓ Одобренное знание → commit в Git ↓ OpenViking индексирует → агенты вспоминают

Как подключить: установите Obsidian → «Open folder as vault» → укажите папку со знаниями (локальную копию репозитория или синхронизированную папку транскриптов). Готово: база знаний агентов — у вас перед глазами.

12. Как учить агентов

Агенты обучаются через безопасный цикл с человеком в центре:

  1. Вы говорите: «запомни: …» / «не делай так» / «исправь».
  2. Долговременный факт агент кладёт в память (memory_store / cold.md).
  3. Урок поведения агент записывает в learnings/inbox/.
  4. По запросу агент предлагает: «из этих уроков стоит сделать правило/навык/запись памяти» — как файл-предложение, не как правку.
  5. Вы одобряете — и только тогда изменение применяется.
  6. Изменение уходит коммитом в Git — у обучения есть история и откат.
Почему так строго? Без саморедактирования по умолчанию: агент, который сам переписывает свои правила, быстро «уезжает» от ваших намерений — поэтому профили прав явно запрещают агентам править CLAUDE.md, правила, навыки и память без одобрения. Агент предлагает — человек решает: это та же логика, что в код-ревью. Секреты не сохраняются никогда, что бы вы ни написали в чате. Правила и навыки меняются аккуратно: маленькими, обратимыми, версионированными шагами.

15. Очередь задач и поведение с медиа

Очередь

  • Мгновенный ак. Каждое сообщение сразу подтверждается: «✅ Task received. Queue position: N» — вы всегда знаете, что задача принята.
  • Очередь на чат. Задачи выполняются по одной, по порядку — можно слать несколько подряд.
  • /queue — показать очередь; /cancel — снять задачи.
  • Таймаут — не тишина. Если задача не уложилась в лимит, придёт «⏱ … Moving on to the next task», и очередь продолжится.
  • Гарантия «без тишины». Любой исход — ответ, ошибка, таймаут — явно сообщается. Молчание = повод писать Uran.
  • Неподдерживаемые файлы получают понятный отказ, а не игнор.

16. Обновления: repo-first деплой

Почему дрейф опасен. Если код правится прямо в продакшне, Git перестаёт быть истиной: следующий деплой молча перезапишет правку, откатиться будет некуда, а отладка превратится в археологию. Поэтому правило одно: изменение рождается в Git и попадает на сервер только через скрипты.

Правильный цикл обновления

cd /root/wayan-claude-install
bash scripts/update.sh
sudo bash scripts/deploy-gateway.sh --dry-run
sudo bash scripts/deploy-gateway.sh --check
sudo bash scripts/deploy-gateway.sh --restart
КомандаЧто делаетМеняет ли что-то
update.shgit pull + полный прогон тестов + проверка дрейфанет только репозиторий
--dry-runпоказывает, что будет сделано при деплоенет
--checkотчёт о дрейфе Git ↔ продакшн: PASS/FAIL, только имена файловнет
--restartбэкап → синхронизация обоих /opt → проверка импорта → перезапуск сервисов → повторная сверкада это и есть деплой

Бэкапы и откат

Каждый деплой автоматически оставляет снапшот: /opt/wayan-*/gateway.bak.ГГГГММДД-ЧЧММСС. Откат при проблеме: остановить сервис → вернуть папку из бэкапа → запустить → проверить статус. Деплой-скрипт по построению не трогает env-файлы, секреты, OpenViking и рабочие пространства.

⚠️ Никогда не правьте напрямую: /opt/wayan-*/gateway (код — только через деплой), содержимое бэкапов, и не редактируйте «на сервере по-быстрому» то, что должно жить в Git. Env-файлы — единственное, что правится прямо на сервере (это секреты, им в Git нельзя).

17. Безопасность и права

МеханизмЧто это и зачем
Ролевые праваУ каждого агента свой .claude/settings.json: read-only диагностика и ресёрч разрешены, опасное — закрыто. Jupiter — рабочий без админ-прав; Uran дополнительно может перезапускать сервисы wayan (и только их) через узкий sudoers.

Запрет саморедактированияDeny-правила не дают агенту править собственные CLAUDE.md, правила, навыки, память. Обучение — только через предложения и ваше одобрение (раздел 12).
Защита секретовТокены и ключи — только в /etc/wayan-*.env и ov.conf с правами 600; .gitignore блокирует их путь в Git; агентам запрещено светить значения в логах и чатах.
OpenViking — только localhostПамять слушает 127.0.0.1:1933 и недоступна из интернета: в ней — ваши знания, наружу её нельзя.
Закрытый порт 8080 (M6, фаза 1)Публичный вебхук-порт легаси-бота закрыт: бот переведён на localhost, правило файрвола удалено. Меньше открытых портов — меньше поверхность атаки.

18. Если что-то сломалось

19 типовых проблем. Откройте свою карточку: симптом → причина → лечение → команда проверки.

SSH: пароль не подходит / ключ не работает

Симптом: Permission denied (publickey,password). Причина: опечатка, не тот IP, ключ не скопирован на сервер.

Лечение: проверьте IP в кабинете провайдера; пароль вводится вслепую — это нормально; ключ: ssh-copy-id root@SERVER_IP.

ssh -v root@SERVER_IP
git pull не проходит

Симптом: conflict / divergent branches / local changes would be overwritten. Причина: в клоне на сервере что-то правили руками.

Лечение: посмотрите, что изменилось; нужное перенесите в коммит, мусор отбросьте.

cd /root/wayan-claude-install && git status && git stash && git pull
install.sh: permission denied / must run as root

Симптом: отказ при запуске. Причина: нет sudo или файл не исполняемый.

Лечение: запускайте с sudo; при необходимости chmod +x install.sh или sudo bash install.sh.

sudo bash install.sh
Telegram-бот молчит (токен)

Симптом: бот не отвечает вообще. Причина: пустой/неверный токен, токены перепутаны местами, сервис не перезапущен после правки env.

Лечение: сверьте токен с BotFather, поправьте env, перезапустите сервис.

sudo systemctl restart wayan-jupiter.service && journalctl -u wayan-jupiter.service -e
Claude не залогинен

Симптом: агент отвечает ошибкой на каждое сообщение; в логах — auth-ошибки Claude. Причина: логин не выполнен или выполнен под root.

Лечение: залогиньтесь именно под wayan (шаг 7) и перезапустите сервисы.

sudo -u wayan -i claude
Голос не расшифровывается (Groq)

Симптом: «voice is disabled» или ошибка на голосовое. Причина: нет GROQ_API_KEY, ключ невалиден, сервис не перезапущен.

Лечение: проверьте ключ в env и баланс/лимиты в console.groq.com, перезапустите сервис.

grep -c GROQ_API_KEY /etc/wayan-jupiter.env && sudo systemctl restart wayan-jupiter.service
OpenViking health не отвечает

Симптом: curl на /health молчит или ошибка. Причина: контейнер не запущен / Docker не стартовал.

Лечение: поднимите compose, посмотрите логи контейнера.

cd /home/wayan/.openviking && docker compose up -d && docker compose logs --tail 50
memory_store падает

Симптом: агент говорит, что не смог сохранить в память. Причина №1: баланс OpenAI пуст (нужно ≥ $5); либо OpenViking не запущен; либо ключ в ov.conf неверный.

Лечение: пополните баланс OpenAI; проверьте /health; проверьте права ov.conf (600, wayan:wayan).

curl -s http://127.0.0.1:1933/health
Docker не установлен

Симптом: docker: command not found. Причина: Docker не входит в базовый установщик Wayan — он нужен только для OpenViking.

Лечение: установите Docker Engine + compose-плагин по официальной инструкции docs.docker.com для Ubuntu, затем вернитесь к шагу 10.

docker --version && docker compose version
ffmpeg отсутствует

Симптом: агент пишет, что визуальный анализ недоступен, разобрал только звук. Причина: ffmpeg не установлен.

Лечение: установить и перезапустить сервис.

sudo apt-get install -y ffmpeg && sudo systemctl restart wayan-jupiter.service

Симптом: в ответе нет блока про картинку. Причина: VIDEO_VISUAL_ANALYSIS выключен (по умолчанию false).

Лечение: включите флаг в env обоих (или нужного) агентов и перезапустите.

20. Репозиторий как шаблон: соберите свою систему

Wayan Agents — открытый шаблон. Любой может взять его за основу собственной агентской системы:

  1. Форкните репозиторий на GitHub — у вас своя копия с историей.
  2. Выберите пакет (WAYAN_PACK) как стартовую специализацию — или напишите свой пакет по docs/PACKS.md.
  3. Кастомизируйте шаблоны: templates/*/CLAUDE.md и USER.md — имена, роли, тон ваших агентов.
  4. Кастомизируйте навыки в skills/ — плейбуки под ваши задачи.
  5. Кастомизируйте оркестрацию: правила, карту сервисов, заготовку памяти в orchestration/.
  6. Секреты — только локально: env-файлы, ov.conf, accounts.md никогда не коммитятся (это уже закрывает .gitignore).
  7. Git — для документов и кода, не для рантайм-секретов и не для сырых медиа.
  8. Obsidian — для знаний: ваша Markdown-база растёт вместе с агентами и остаётся вашей.
  9. Результат — ваш собственный бренд и агентская система: «Jupiter и Uran» легко становятся любыми персонажами вашего мира.
